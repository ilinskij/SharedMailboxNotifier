# Shared Mailbox Notifier

VSTO-надстройка для Outlook, которая показывает уведомления Windows Toast при получении новых писем в общих (shared) почтовых ящиках.

[English](README.md) · **Русский**

## Проблема

Outlook не показывает уведомления на рабочем столе для общих (дополнительных) почтовых ящиков — только для собственной учётной записи пользователя. Если вы работаете с несколькими общими ящиками (например, `info@`, `support@`, `hr@`), приходится проверять их вручную.

## Что делает надстройка

При поступлении нового непрочитанного письма в папку «Входящие» любого отслеживаемого общего ящика надстройка показывает стандартное уведомление Windows Toast, содержащее:

- **Тему, имя отправителя и превью текста письма**
- **Фото отправителя** (из Active Directory или локальных контактов)
- **Кнопки действий**: Ответить, Прочитано, Назначить категорию
- **Имя почтового ящика и время** в строке атрибуции

Уведомления автоматически убираются при прочтении или удалении письма в Outlook.

## Возможности

- **Два режима мониторинга**: только общие ящики или все ящики (с автоматическим отключением встроенных уведомлений Outlook во избежание дублирования)
- **Toast-уведомления** в Windows 10/11 с интерактивными кнопками
- **Fallback на всплывающие подсказки** (BalloonTip) в Windows 7/8
- **Назначение категории** прямо из уведомления (с настраиваемым выбором до 5 категорий)
- **Фото отправителей** из Exchange/AD (через свойства MAPI) и опционально из локальных контактов
- **Динамическое отслеживание хранилищ**: автоматически подхватывает ящики, добавленные или удалённые во время работы
- **Локализация**: английский, немецкий, русский, украинский
- **Страница настроек**, встроенная в диалог параметров Outlook
- **Запись в журнал событий** Windows для диагностики

## Требования

- Windows 7 или новее (Windows 10 1607+ для Toast-уведомлений)
- Microsoft Outlook 2010, 2013, 2016, 2019 или Microsoft 365 (настольная версия)
- .NET Framework 4.8
- Visual Studio 2022 с рабочей нагрузкой **Разработка надстроек для Office и SharePoint** (для сборки из исходников)

## Сборка

1. Откройте `SharedMailboxNotifier.csproj` в Visual Studio 2022.
2. Убедитесь, что установлена рабочая нагрузка **Разработка надстроек для Office и SharePoint**.
3. Соберите решение. NuGet-пакеты восстановятся автоматически.
4. Нажмите F5 для отладки — Outlook запустится с загруженной надстройкой.

## Установка

Для развёртывания конечным пользователям опубликуйте через ClickOnce из Visual Studio или создайте MSI-установщик. Надстройка требует Visual Studio 2010 Tools for Office Runtime (VSTO Runtime), который включён в Office 2010+.

## Настройка

После установки откройте Outlook → **Файл → Параметры → вкладка «Надстройки» → Shared Mailbox Notifier**.

| Параметр | Описание |
|---|---|
| **Режим мониторинга** | *Только общие* — отслеживает дополнительные ящики, Outlook обрабатывает уведомления для личных. *Все ящики* — отслеживает всё, встроенные уведомления Outlook отключаются. |
| **Звуковое уведомление** | Включить или отключить звук уведомления. |
| **Круглая иконка** | Обрезать фото отправителя / логотип приложения до круга в уведомлениях. |
| **Поиск фото в контактах** | Искать фото отправителей в локальных контактах (может замедлить уведомления при большой адресной книге). |
| **Настроить категории** | Выбрать до 5 категорий Outlook для отображения в выпадающем списке уведомления. |

## Технические детали

- Обнаружение новых писем основано на событии `Items.ItemAdd` папки «Входящие». Объектная модель Outlook не предоставляет отдельного события «новое письмо доставлено в это хранилище» для общих ящиков, поэтому `ItemAdd` — единственный надёжный механизм. Побочный эффект: любой непрочитанный элемент, появившийся во «Входящих» — будь то обычная доставка, перенос из «Нежелательной почты» или перетаскивание из другого ящика — вызовет уведомление. Это осознанный компромисс, а не ошибка.
- Надстройка работает в процессе Outlook на STA-потоке. Все обработчики COM-событий (`ItemAdd`, `StoreAdd`, `BeforeStoreRemove`) маршалируются COM на этот поток.
- Ссылки на COM-объекты (`Items`, `Stores`, `Folder`) удерживаются явно для предотвращения сборки мусора, которая бы тихо отписала обработчики событий.
- Обратные вызовы Toast-уведомлений приходят на MTA-поток. `ToastActionHandler` получает свежую ссылку на `Outlook.Application` через `Marshal.GetActiveObject`, а не использует привязанную к STA ссылку.
- Настройки хранятся в `HKCU\Software\SharedMailboxNotifier`. Надстройка также управляет собственными ключами реестра Outlook (`NewmailDesktopAlerts`, `PlaySound`) для предотвращения дублирования уведомлений в режиме «Все ящики».
